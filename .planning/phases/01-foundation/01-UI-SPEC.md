# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Scope

Phase 1 delivers **no new UI surfaces**. It is a pure infrastructure phase:
- Database migration (add `is_active` and `display_name` columns to `members` table)
- Server utility verification (`createAuthUser()` wrapper around `auth.admin.createUser()`)
- `requireMember()` utility updated to reject `is_active = false`

No pages, components, or visual elements are added or modified in this phase.

This contract documents the **baseline design system** that all subsequent phases inherit. Executors working on Phase 1 must not deviate from these established tokens when touching any server utility output (error messages, HTTP status codes) that will surface in the UI.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `px-2`, `py-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`) |
| lg | 24px | Section padding (`px-6`, `py-6`, `gap-6`) |
| xl | 32px | Layout gaps (`gap-8`, `mb-8`) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level spacing (`py-16`) |

Exceptions:
- Sidebar nav items: `px-2.5 py-1.5` (10px / 6px) — matches existing dashboard.vue pattern, kept for visual consistency
- Touch targets (mobile): minimum 44px height for all interactive controls
- Card gaps in member list: `space-y-3` (12px) — matches existing members.vue pattern

## Typography

Font family: **DM Sans** for both body and headings (single font, varies by weight only).

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|---------|
| Body | 14px | 400 | 1.5 | `text-sm` |
| Label / UI | 13px | 500 | 1.4 | `text-[13px] font-medium` |
| Heading | 18px | 600 | 1.3 | `text-lg font-semibold` |
| Page title | 24px | 700 | 1.2 | `text-2xl font-bold` |

Notes:
- 13px label size is the existing dashboard sidebar pattern — preserve it
- Two weights in use: regular (400) and semibold/medium (500–600). Bold (700) used only for brand logo ("PoinSaku")
- All text must maintain 4.5:1 contrast minimum in both light and dark mode
- Source: existing dashboard.vue and members.vue patterns

## Color

| Role | Value | Tailwind / CSS Var | Usage |
|------|-------|-------------------|-------|
| Dominant (60%) | White / slate-900 dark | `bg-(--ui-bg)` | Page background, surfaces |
| Secondary (30%) | White / gray-900 dark | `bg-white dark:bg-gray-900`, `glass-card` | Cards, sidebar, nav background |
| Accent (10%) | Sky #0ea5e9 | `text-primary`, `bg-primary`, `text-primary-500` | (see below) |
| Destructive | Red (Nuxt UI `error`) | `color="error"` | Destructive actions only |

Accent (`primary` / sky) is reserved for:
- Active navigation item background (`bg-primary-500/10`) and text (`text-primary-500`)
- Primary action buttons (`UButton` default variant)
- Brand logo text
- Icon container backgrounds (`bg-primary-500/10`)
- Active state indicators (badges, selected states)

Accent is NOT used for:
- Body text
- Card borders
- Non-interactive decorative elements
- Secondary/cancel buttons (use `variant="outline"`)

Semantic badge colors (source: existing members.vue):
- Owner role badge: `color="error"`
- Admin role badge: `color="warning"`
- Cashier/neutral role badge: `color="neutral"`
- Active status badge: `color="success"` (to be added in Phase 3)
- Inactive status badge: `color="error" variant="soft"` (to be added in Phase 3)

## Copywriting Contract

All copy is in **Bahasa Indonesia** (established by existing pages). Phase 1 introduces one new user-facing behavior: rejection of inactive accounts.

| Element | Copy (Indonesian) | Context |
|---------|-------------------|---------|
| Inactive account rejection | "Akun Anda telah dinonaktifkan. Hubungi pemilik bisnis." | HTTP 403 from `requireMember()` when `is_active = false` |
| Auth admin error (generic) | "Gagal membuat akun staf. Coba lagi." | `createAuthUser()` error fallback surfaced in Phase 2+ |
| Auth admin error (email exists) | "Email sudah terdaftar. Gunakan email lain." | `createAuthUser()` duplicate email error |

Notes:
- Error copy follows `e.data?.message || e.message` pattern already in members.vue — server must return structured `{ message: '...' }` in error body
- No empty state copy needed for Phase 1 (no list UI)
- No destructive confirmation UI in Phase 1 (no delete actions)

## Interaction Contracts

Phase 1 has no interactive UI. The following contracts apply to server-side behavior that surfaces in the UI:

| Behavior | Contract |
|----------|----------|
| `requireMember()` rejects inactive member | Returns HTTP 403 with `{ message: 'Akun Anda telah dinonaktifkan. Hubungi pemilik bisnis.' }` |
| `requireMember()` check ordering | Check auth → check member row exists → check `is_active` → check scope. Active check before scope resolution. |
| `createAuthUser()` error shape | Throws `createError({ statusCode, message })` — same pattern as all other server utilities |
| Loading state | Spinner: `i-lucide-loader-2 animate-spin` — do not introduce new loading patterns |
| Toast notifications | Use `useToast()` with `color: 'success'/'error'` and `icon: 'i-lucide-check'/'i-lucide-alert-circle'` |

---

## Anti-Patterns (Do NOT Use)

Source: design-system/poinsaku/MASTER.md — enforced across all phases.

- Emojis as icons — use Lucide SVG icons (`i-lucide-*`)
- Solid opaque backgrounds on cards — use `glass-card` or Nuxt UI `UCard`
- Glass without vibrant background behind it
- Scale transforms on hover (layout-shifting)
- Invisible focus states
- Transitions faster than 150ms or slower than 300ms
- Raw UUIDs displayed in UI — always resolve to human-readable label
- `confirm()` for destructive actions on mobile — use modal with explicit confirm button (Phase 3)
