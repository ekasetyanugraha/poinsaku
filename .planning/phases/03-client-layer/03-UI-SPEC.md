## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, row gaps within cards |
| md | 16px | Default field spacing, card padding, staff list card row internal spacing (`space-y-4`) |
| lg | 24px | Section gaps, modal body padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Empty state vertical padding (`py-12`) |
| 3xl | 64px | Full-page vertical centering contexts |

Exceptions:
- Touch targets for icon-only action buttons (three-dot menu trigger): minimum 44px height via `size="sm"` + padding (Nuxt UI handles this)
- Sidebar width: fixed 224px (w-56) — inherits from `dashboard.vue` layout

Source: Existing `members.vue`, `dashboard.vue` layout patterns — pre-populated, not defaulted.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Staff list secondary info, muted labels, badge text, sidebar nav items, form field labels, compact UI text |
| Heading | 18px (text-lg) | 600 (semibold) | 1.2 | Page title "Anggota Tim", section headings, modal titles |
| Display | — | — | — | Not used in this phase |

Notes:
- Exactly 2 declared weights: 400 (regular) and 600 (semibold). Weight 500 (medium) is NOT a declared scale entry — the project loads DM Sans 400 and 600 only.
- Label-level text (sidebar nav, form labels, compact UI text at 12px) uses `text-xs` (12px) at weight 400 — it is a size variant of Body, not a separate weight tier.
- All text in Bahasa Indonesia — no English labels anywhere in rendered UI
- Error and success inline messages: 14px, weight 400, color from semantic tokens (red-500, success)
- `font-heading` class uses DM Sans (same as `font-sans`) per `main.css` `@theme` block
- Page subtitle / description text: 14px, weight 400, `text-(--ui-text-muted)`

Source: Detected from `members.vue` (`text-lg font-semibold`), `login.vue` (`text-sm`), `dashboard.vue` (`text-[13px] font-medium`), `main.css` font definitions. Label size corrected from 13px to 12px per checker review (creates visible hierarchical step from 14px body).

## Color

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `--ui-bg` / white in light, gray-950 in dark | `bg-(--ui-bg)` | Page background, content area (`bg-gray-50 dark:bg-gray-950`) |
| Secondary (30%) | white / gray-900 with glass overlay | `glass-card`, `bg-white dark:bg-gray-900` | Staff list cards, sidebar, header, modals, slideover |
| Accent (10%) | Sky #0ea5e9 | `text-primary`, `bg-primary-500/10`, `text-primary-500` | Primary CTA buttons only, active nav item highlight, "Tambah Staff" button, loading spinner color, UBadge `color="primary"` for owner "Pemilik" badge |
| Destructive | Red — `color="error"` | `bg-red-500/10 text-red-500` | Delete action in dropdown, error inline message banner, deactivate confirmation modal confirm button |
| Success | Green — `color="success"` | Nuxt UI success token | Active status badge `UBadge color="success"`, toast on successful operations |
| Neutral | Slate | `color="neutral"` | Inactive status badge, scope badge, secondary action buttons (Batal) |

Accent reserved for: Primary CTA buttons (Tambah Staff, Masuk), active nav state, owner "Pemilik" badge only.
Do NOT use accent for: informational badges, hover states on non-primary actions, status badges (use success/neutral/error for those).

Glass utilities used in this phase:
- `glass-card` — all staff list UCard rows, slideover card content, modal content
- `glass-sidebar` — inherited from `dashboard.vue` layout (no change)
- `glass` — NOT used on staff list cards (glass-card is correct for dashboard cards)

Source: `app.config.ts` (primary: sky, neutral: slate), `main.css` glass tokens, `members.vue` existing badge patterns, `MASTER.md`.

## Interaction Contracts

### Staff List Loading State
- Show centered `UIcon name="i-lucide-loader-2" class="size-7 animate-spin text-(--ui-text-muted)"` while `status === 'pending'`
- Skeleton layout: NOT used — spinner only (Claude's discretion area; spinner is simpler and consistent with existing members.vue pattern)

### Staff List Empty State
- Trigger: list loads successfully, zero staff rows returned (owner has no staff yet)
- Layout: centered in list area, padding `py-12`
- Icon: `i-lucide-users` at `size-12 text-(--ui-text-muted) mx-auto mb-3`
- Heading: "Belum ada staff" — `text-sm font-medium`
- Body: "Tambahkan staff pertama dengan menekan tombol di atas." — `text-sm text-(--ui-text-muted)`

### Slideover (Staff Creation)
- Width: 420px (`w-[420px]`) on desktop; full-width on mobile (Nuxt UI handles this)
- Opens from right — `side="right"`
- Title: "Tambah Staff" in slideover header
- Form layout: vertical `space-y-4`, labels above inputs
- Scope field: USelect with placeholder "-- Pilih cakupan --", NO pre-selected value. Scope ID field hidden until scope type is chosen.
- Submit button: "Buat Akun Staff" — bottom of form, primary color, full-width inside button group
- Cancel: "Batal" ghost variant alongside submit
- On success: close slideover + toast "Staff berhasil dibuat" (success, `i-lucide-check`)
- On error: inline toast "Gagal membuat staff: [server message]" (error, `i-lucide-alert-circle`)

### Per-Row Dropdown Actions
- Trigger: `UButton variant="ghost" size="sm" icon="i-lucide-ellipsis-vertical" aria-label="Aksi staff"` — right side of card
- Hidden entirely on owner row (no element rendered, not just disabled)
- Items for active staff: Reset Password | Pindah Cabang | --- | Nonaktifkan | Hapus
- Items for inactive staff: Reset Password | Pindah Cabang | Aktifkan | --- | Hapus
- "Nonaktifkan" and "Hapus" are destructive — rendered with `color: 'error'` in items config
- Divider before destructive group via nested array structure

### Confirmation Modals (Destructive Actions)
- Delete confirmation title: "Hapus Staff"
- Delete body: "Hapus staff **[display_name || email]**? Tindakan ini tidak dapat dibatalkan."
- Delete confirm button: "Hapus" — `color="error"`
- Delete cancel button: "Batal" — `variant="outline"`
- Deactivate confirmation title: "Nonaktifkan Staff"
- Deactivate body: "Nonaktifkan akun **[display_name || email]**? Staff tidak akan bisa login sampai diaktifkan kembali."
- Deactivate confirm button: "Nonaktifkan" — `color="error"`

### Password Reset Modal
- Title: "Reset Password"
- Body: "Password baru untuk **[display_name || email]**:"
- Single `UInput type="password"` field, `placeholder="Password baru"`
- Confirm button: "Simpan Password" — primary color
- Cancel: "Batal" — `variant="outline"`
- On success: modal closes + toast "Password berhasil diubah" (success)

### Branch Reassignment Modal
- Title: "Pindah Cabang"
- Body: Current branch displayed as read-only label ("Cabang saat ini: [branch name]")
- `USelect` for new branch — placeholder "Pilih cabang baru"
- Confirm button: "Pindahkan" — primary color
- Cancel: "Batal" — `variant="outline"`
- On success: modal closes + toast "Cabang berhasil diubah" (success)

### Staff Login Interaction
- No page decorations — plain white/dark card centered on `bg-(--ui-bg)` (no gradient orbs, no blur orbs)
- Title: "Masuk sebagai Staff" — `text-lg font-semibold text-center`
- No "Daftar" link, no "Lupa password" link — stripped to functional minimum
- Error for wrong credentials: "Email atau password salah."
- Error for deactivated account: "Akun Anda telah dinonaktifkan. Hubungi pemilik bisnis."
- Both errors shown in `bg-red-500/10 text-red-500 rounded-md p-3` inline banner (identical pattern to `login.vue`)
- Submit button shows `:loading` spinner during auth request
- After successful login: loading spinner persists until redirect completes

### Transitions
- All state changes use Nuxt UI's built-in transition system
- Slideover: default Nuxt UI enter/leave animation (translate-x from right)
- Modal: default Nuxt UI fade + scale animation
- Inactive row opacity: `transition-opacity duration-200`
- Dropdown open/close: Nuxt UI handles internally

## Accessibility

- All interactive elements have `cursor-pointer` (enforced in `main.css` `@layer base`)
- Touch targets: 44px minimum height for action buttons (Nuxt UI `size="sm"` meets this)
- Icon-only action button: `aria-label="Aksi staff"` declared on the three-dot `UButton` trigger
- Focus states: visible via Nuxt UI defaults (ring on focus)
- `prefers-reduced-motion`: respected via `main.css` media query (disables all animations)
- Color contrast: 4.5:1 minimum — `text-(--ui-text-muted)` on white exceeds threshold
- Destructive confirmation modals: require explicit button press (no keyboard shortcut that bypasses)
- Staff login page: `autocomplete="email"` and `autocomplete="current-password"` on inputs
