## Spacing Scale

Declared values (multiples of 4). All values already in use across the cashier page.

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps (`gap-1`), inline icon margins (`mr-1`) |
| sm | 8px | Compact element spacing (`gap-2`), input row gaps |
| md | 16px | Default element spacing (`space-y-4`, `p-4`) |
| lg | 24px | Section padding (`space-y-6`), card inner gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Not used in cashier layout (mobile-focused) |
| 3xl | 64px | Not used in cashier layout (mobile-focused) |

Exceptions:
- Phone input touch target: minimum 48px height (`h-12`) — nearest multiple of 4 above the 44px accessibility minimum; mobile cashier use case, finger-operated
- +/- stamp counter buttons: `size="sm"` from @nuxt/ui, rendered at ~32px — acceptable for tap target with surrounding whitespace
- Camera viewfinder: `aspect-square max-w-sm` — not on the 8-point scale; it is a viewport-relative block

Two weights only: `font-normal` (400) for body and label text; `font-semibold` (600) for headings and display text.

| Role | Size | Weight | Line Height | Usage |

| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Phone number under customer name, program description, stamp count, preview text |

| Role | Token | Usage |
|------|-------|-------|

5. "Mode Kasir" header icon and label

Accent is NOT applied to: neutral navigation (ghost buttons), secondary actions (outline buttons), card borders, phone input.

## Screen States

This phase extends the existing state machine. Each state has a distinct layout region.

### State: `scan` (scan section — unchanged except manual entry replacement)

**Phone lookup section replaces the existing "Atau masukkan kode manual" block.**

Primary visual anchor: the phone input row (`UInput` + `UButton "Cari Pelanggan"`) is the focal point — it is the only interactive element below the QR scanner in this state.

Layout (below the QR scanner):
```
[ divider line ]
[ label: "Atau cari pelanggan manual:" ]
[ UInput phone h-12 ] [ UButton "Cari Pelanggan" ]
[ error banner (conditional) ]
```

- Input: `type="tel"`, `class="h-12"`, placeholder `"Contoh: 08123456789"`, icon `i-lucide-phone`
- Button: default `@nuxt/ui` primary fill, label "Cari Pelanggan", `size="md"`
- Loading: button shows `:loading="phoneLoading"` spinner while fetch is in-flight
- Error banner: `class="flex items-center gap-2 rounded-lg bg-red-500/10 p-3 text-sm text-red-500"` — same pattern as existing `scanError` block (line 45-48 in cashier/index.vue)

### State: `program-select` (NEW intermediate state)

Shown when lookup returns 2+ active stamp programs for the customer at this business.

Primary visual anchor: the list of program picker cards. The customer name and phone serve as context above the list; the list items are the actionable focal point.

Layout:
```
[ UButton ghost "← Scan Baru" ]
[ Heading: "Pilih Program Stempel" ]
[ customer name + phone (text only — no card needed) ]
[ List of program picker cards ]
```

Program picker card (one per program):
- Element: `UButton` styled as a full-width card-like item — `variant="outline"`, `class="w-full text-left justify-start cursor-pointer"`, `size="lg"`
- Content: program name (semibold) on line 1, current stamp count on line 2 (`text-sm text-muted`)
- No UCard wrapper needed — button list with gap-2 is sufficient

### State: `customer` (unchanged visual structure)

The existing customer info card and stamp action card are reused without visual changes. The `handleAddStamp()` function is called identically.

**Amount-based stamp mode addition** (new sub-UI within the stamp action card):

When `stamp_config.stamp_mode === 'amount_based'`, replace the +/- counter with:
```
[ UFormField label="Nominal Transaksi (Rp)" ]
  [ UInput type="number" placeholder="50000" v-model.number="txAmount" ]
[ preview text: "Rp 50.000 = 5 stempel" ]  ← shown only when calculatedStamps > 0
[ UButton "Tambah Stempel" primary fill full-width ]
```

Preview text style: `class="text-sm text-muted"` — same pattern as existing cashback preview (line 148-151 in cashier/index.vue).

## Interaction Contract

### Phone Input Behavior

- Input type: `type="tel"` — triggers numeric keyboard on mobile (no masking library needed)
- Input height: `class="h-12"` — 48px touch target for finger-operated mobile use
- Validation: on submit only (not on-change) — match existing cashier page pattern
- Submit trigger: "Cari Pelanggan" button click OR Enter key on input (`@keyup.enter="handlePhoneLookup"`)
- Empty input: "Cari Pelanggan" button is `:disabled="!phoneSearch.trim()"` — no error shown, button is greyed
- While loading: "Cari Pelanggan" button shows `:loading="phoneLoading"` — input remains editable
- After error: error banner appears below the input row — input is NOT cleared (cashier may want to edit)
- After success (1 program): transition directly to `'customer'` state — phone input becomes irrelevant
- After success (2+ programs): transition to `'program-select'` state

### Program Picker Behavior

- Clicking a program card: sets `verifiedData` with that program's data shape and transitions to `'customer'` state
- Back from program picker: `resetState()` — returns to `'scan'` state, clears phone input

### Amount-Based Stamp Input

- Input type: `type="number"` — matches existing `txAmount` input pattern (line 146)
- Preview: computed value updates reactively as amount changes — shown only when `calculatedStamps > 0`
- Submit: disabled when `txAmount <= 0` — matches existing cashback button pattern (line 155)
- Rounding: `Math.floor(txAmount / amount_per_stamp)` — server does the authoritative calculation; preview is informational only

### Reset After Stamp Add

Per locked decision (CONTEXT.md): after successful stamp add, call `resetState()`. The new `phoneSearch` ref must be added to `resetState()` — cleared to empty string. `lookupCustomer`, `lookupPrograms`, `stampConfig` refs also cleared.

## Accessibility Notes

- Phone input must have an explicit label or `aria-label` — use `UFormField` label or `aria-label="Nomor HP pelanggan"`
- Program picker buttons must be keyboard navigable (UButton default behavior — no additional work)
- Error banners must include an icon + text (pattern already established in cashier page line 45-48)
- Stamp counter +/- buttons already have `aria-label="Kurangi stempel"` / `aria-label="Tambah stempel"` — preserve these
- `prefers-reduced-motion` already handled globally in `main.css` — no new animations needed for this phase
