<task type="auto">
  <name>Task 1: Configure Nitro cloudflare_pages preset and create wrangler.toml</name>
  <files>nuxt.config.ts, wrangler.toml, package.json</files>
  <action>
1. In `nuxt.config.ts`, add the Nitro preset configuration:
   ```ts
   nitro: {
     preset: 'cloudflare_pages',
   },
   ```
   Place it after the existing `vite` block.

2. Create `wrangler.toml` at project root with:
   ```toml
   name = "stampku"
   compatibility_date = "2024-09-23"
   compatibility_flags = ["nodejs_compat"]
   pages_build_output_dir = "dist"

   [vars]
   NUXT_PUBLIC_APP_URL = "https://stampku.pages.dev"
   ```
   - `nodejs_compat` is required because `jsonwebtoken` and `@nuxtjs/supabase` use Node.js built-in modules internally.
   - `pages_build_output_dir` must be `dist` — that is the Nuxt 4 / Nitro cloudflare_pages output directory.
   - Do NOT put secrets (SUPABASE_KEY, service role key, etc.) in wrangler.toml — those go in Cloudflare dashboard or via `wrangler secret`.

3. In `package.json`, add deploy-related scripts:
   ```json
   "build:cf": "NITRO_PRESET=cloudflare_pages nuxt build",
   "deploy": "wrangler pages deploy dist",
   "preview:cf": "wrangler pages dev dist"
   ```

4. Add `wrangler.toml` to the project (it should NOT be in .gitignore since it contains no secrets — only public config and compatibility flags).
  </action>

  </verify>

  <files>server/api/qr/generate.post.ts, server/api/vouchers/generate.post.ts, server/api/wallet/google.post.ts, server/api/wallet/samsung.post.ts, .env.example</files>

1. Update `import crypto from 'crypto'` to `import crypto from 'node:crypto'` in:
   - `server/api/qr/generate.post.ts` (line 2)
   - `server/api/vouchers/generate.post.ts` (line 1)

   The `node:` prefix is the standard ESM import form and is required by Cloudflare Workers' `nodejs_compat` mode to resolve Node.js built-ins correctly. Without the prefix, Workers may fail to resolve the module.

2. Update `import jwt from 'jsonwebtoken'` to `import jwt from 'jsonwebtoken'` — no change needed here. The `jsonwebtoken` package is an npm dependency (not a Node.js built-in), so it resolves normally. Its internal use of `crypto` is handled by the `nodejs_compat` flag.

