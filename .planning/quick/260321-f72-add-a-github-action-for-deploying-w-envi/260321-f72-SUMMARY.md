# Quick Task Summary: Add GitHub Action for Deploying

## What was done

Created `.github/workflows/deploy.yml` — a GitHub Actions workflow that automatically deploys to Cloudflare Pages on push to `main`.

### Workflow details

- **Trigger:** `push` to `main` branch
- **Runner:** `ubuntu-latest`
- **Steps:** checkout → setup Node 20 → npm ci → build:cf → wrangler pages deploy
- **Auth:** Uses `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` secrets
- **Build env vars:** All 17 environment variables from `.env.example` passed via GitHub Secrets

### GitHub Secrets required

| Secret | Purpose |
|--------|---------|
| `CLOUDFLARE_API_TOKEN` | Cloudflare API token (Pages edit permission) |
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare account ID |
| `SUPABASE_URL` | Supabase project URL |
| `SUPABASE_KEY` | Supabase anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key |
| `NUXT_PUBLIC_APP_URL` | Production app URL |
| `NUXT_QR_TOKEN_SECRET` | QR code token signing secret |
| `NUXT_APPLE_*` (6 secrets) | Apple Wallet pass signing |
| `NUXT_GOOGLE_*` (3 secrets) | Google Wallet integration |
| `NUXT_SAMSUNG_*` (3 secrets) | Samsung Wallet integration |
